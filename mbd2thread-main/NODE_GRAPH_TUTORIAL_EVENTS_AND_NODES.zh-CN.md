# 节点图教程：事件规则与节点用途（新人向 / 不用 KJS）

本文专门回答两件事：

1. **每个常用节点是干什么的、怎么接线、常见坑是什么**  
2. **有哪些事件能用、该选哪个事件、可取消事件怎么理解**

说明：本教程不涉及 KubeJS（不教写脚本），只用编辑器里“机器 / 逻辑 / 值 / Minecraft 数据 / 工具”等内置节点。

---

## 一、事件：规则与思路

### 1) 事件图是什么

事件图就是“当某个事件发生时，自动执行的一张图”。你在右侧事件列表里添加/编辑的每一项，背后都对应一张事件图。

### 2) 事件图的三条硬规则

1. **只有被触发的事件图才会运行**：你把图连得再漂亮，事件不触发就不会执行。
2. **触发线决定执行顺序**：黄色线从“触发开始 → 已触发”出发，连到需要按顺序运行的节点（写入/动作节点）。
3. **数据线必须接全**：写入类节点（例如“复合标签写入”“设置自定义数据”“设置机器状态”）除了触发线，还必须把数据输入端口接满，否则触发了也不会产生效果。

### 3) 可取消事件是什么（为什么会有“取消”参数）

部分事件在 Forge 层面是“可取消”的（Cancelable）。MBD2 会在这类事件里额外提供一个可写参数：

- **取消（Boolean）**：写 true 等价于 `event.setCanceled(true)`，意味着“阻止本事件继续产生效果”。

你不需要理解代码层，只要记住：

- **“取消”不是给你看的状态**，而是一个“开关”：你写 true，就能把某些行为拦住或中断。

### 4) 事件怎么选：按目的选事件，而不是按名字猜

新人最容易犯的错是“想控制配方，却只在右键事件里改外观状态”。正确思路是：

- **想记住一个开关/模式** → 选“机器放置/右键/打开 UI”等用户交互事件，去写 customData。
- **想阻止配方开始** → 选“配方开始前事件（可取消）”，写 取消=true 来拦截开始。
- **想立刻停止正在运行的配方** → 选“配方开始事件（可取消）”，写 取消=true 来中断运行。
- **想做“配方结束后奖励/清理”** → 选“配方完成事件/配方结束后事件”。

### 5) 本项目可用的常见机器事件（中文名）

以下事件名以本项目中文显示为准（右侧事件列表里看到的）：

- 机器Tick事件：每 tick（服务器）触发，适合定时检查/自动化逻辑（注意性能）。
- 机器客户端Tick事件：客户端 tick（通常用于显示/粒子等）。
- 机器加载事件：机器加载进世界时触发（含区块加载）。
- 机器放置事件：放下机器时触发。
- 机器移除事件：机器被移除时触发。
- 机器打开界面事件 / 机器界面事件：打开 UI 或 UI 交互相关。
- 机器右键事件：玩家右键机器时触发。
- 机器邻接变化事件：邻居方块变化时触发。
- 机器掉落事件：掉落生成前后。
- 机器状态变化事件：机器状态机状态改变时触发（old/new）。
- 自定义数据更新事件：customData 变更时触发（old/new）。
- 配方状态变化事件：配方逻辑状态改变（IDLE/WORKING/WAITING/SUSPEND）。
- 配方修改事件（前/后）：配方被修改前后（用于条件/倍率/产出调整）。
- 配方开始前事件（可取消）：配方将要开始时触发（拦截开始的最佳位置）。
- 配方开始事件（可取消）：配方工作 tick 中触发（中断运行的最佳位置）。
- 配方结束后事件：配方结束后触发（清理/还原）。
- 配方等待事件：卡条件/缺能量等进入 WAITING 时触发。
- 配方完成事件：完成时触发。
- 工作后消耗输入事件：ConsumeInputsAfterWorking 模式下消耗输入后触发。
- 燃料燃尽事件：燃料配方燃尽时触发。

## 二、节点：用途、怎么连、常见坑

下面按“新人最常用到的节点”分类讲。每个节点都写清楚：用途、输入输出、最小连接、坑点。

### A. 起点与调试

#### 1) 触发开始

- **用途**：事件图执行的起点。
- **输出**
  - 已触发（黄色触发端口）
- **怎么用**
  - 把“已触发”连到你希望执行的动作节点（写入/命令/放置方块/设置自定义数据等）。
- **常见坑**
  - 只接数据线不接触发线，导致节点永远不执行。

#### 2) 控制台日志 / 打印

- **用途**：在调试时显示某条数据当前值（例如 paused、坐标、配方状态）。
- **输入**
  - 控制台日志：有触发 + 输入
  - 打印：只有输入（用于界面预览）
- **怎么用（推荐）**
  - 用“控制台日志”并接上触发线：触发开始.已触发 → 控制台日志.触发；你要看的值 → 控制台日志.in

### B. 机器相关（机器分组）

#### 3) 机器信息

- **用途**：从“机器”对象里取常用信息。
- **输入**
  - 机器
- **输出（常用）**
  - 状态（机器状态名）
  - 配方状态（IDLE/WORKING/WAITING/SUSPEND）
  - 自定义数据（CompoundTag）
  - 世界 / 坐标 / 朝向 等
- **典型用法**
  - 读取自定义数据后，用“复合标签读取”取出某个开关。

#### 4) 设置自定义数据

- **用途**：把一个 CompoundTag 写回机器的 customData（会持久化）。
- **输入**
  - 触发（黄色）
  - 机器
  - 数据（CompoundTag）
- **最小连接**
  - 必须同时接：触发线 + 机器 + 数据
- **常见坑**
  - 把“原始 tag”接回去（导致修改丢失）。应当接“复合标签写入.输出”这种“写入后的新 tag”。

#### 5) 设置机器状态

- **用途**：把机器“状态机状态名”切到指定字符串（外观/光照/渲染可随状态变化）。
- **输入**
  - 触发（黄色）
  - 机器
  - 状态（String）
- **规则**
  - 只有当你的状态机里存在该状态名时才会生效（不存在会被忽略）。
- **典型用法**
  - 将暂停外观切到 `suspend`，工作外观切到 `working`（通常配方逻辑会自动切换）。

### C. NBT（Minecraft 数据分组）

#### 6) 复合标签（Compound）

- **用途**：把输入转换为 CompoundTag；常用来“兜底空值”。
- **输入**
  - 输入（Object）
- **输出**
  - 输出（CompoundTag）
- **典型用法**
  - 机器信息.自定义数据 → 复合标签 → 后续读取/写入节点

#### 7) 复合标签读取（Compound Reader）

- **用途**：从 CompoundTag 里按 key 读取一个值。
- **输入**
  - 标签（CompoundTag）
  - key（可不接，使用内部 key）
- **配置**
  - 类型：BOOL / NUMBER / STRING / COMPOUND / LIST
  - key：例如 `paused`
- **输出**
  - 输出（类型随配置变化）
- **常见坑**
  - tag 里没有该 key 时会读到默认值（BOOL 默认 false），所以必须先确保写入过。

#### 8) 复合标签写入（Compound Writer）

- **用途**：把 key/value 写入到 tag 的副本里，输出一个“新的 tag”。
- **输入**
  - 触发（黄色）
  - 标签（CompoundTag）
  - key（可不接，使用内部 key）
  - value（Object）
- **输出**
  - 输出（CompoundTag，新副本）
- **最小连接**
  - 必须接触发线，否则不会输出新 tag。
- **常见坑**
  - 写入后忘了把“输出”接到“设置自定义数据”，导致写入只停留在节点内部。

### D. 值与逻辑（值/逻辑分组）

#### 9) 布尔值（Bool）

- **用途**：布尔常量，或把输入转换成布尔。
- **输入**
  - 输入（可选）
- **输出**
  - 输出（Boolean）
- **规则**
  - 输入没连线：输出=圆形开关值（常量模式）。
  - 输入有连线：输出=输入转换后的布尔值，开关会同步为当前输出。
- **最常用两种模式**
  - 常量 true：开关点亮，且不接输入
  - 转换：把读出来的值接到输入，让输出变成标准 boolean

#### 10) 异或（Xor）

- **用途**：实现“取反”或“二选一翻转”。
- **输入**
  - a（Boolean）
  - b（Boolean）
- **输出**
  - out（Boolean）
- **典型用法（取反）**
  - `paused = paused XOR true`：把当前 paused 接到 a，把“常量 true”接到 b。
- **常见坑**
  - b 接的是 false（默认），那么 `a XOR false = a`，不会翻转。

#### 11) 选择（Select）

- **用途**：根据条件选择 true/false 两个分支的值（返回 Object）。
- **输入**
  - true（Object）
  - false（Object）
  - condition（Boolean）
- **输出**
  - out（Object）
- **典型用法**
  - 根据 paused 选择不同的字符串状态名，或选择不同的数值参数。

#### 12) 相等（Equal）

- **用途**：判断两个对象是否相等（适合字符串/数字/对象）。
- **输入**：a, b
- **输出**：out（Boolean）
- **典型用法**
  - 判断当前状态是否为某个名字：`状态 == "working"`

### E. 参数（事件参数读写）

#### 13) 参数

- **用途**：读取/写回“事件参数”。
- **两种形态**
  - 读取（Get）：只有输出端口
  - 写入（Set）：只有输入端口
- **最常见的可写参数**
  - 取消：出现在可取消事件中，写 true 会拦截/中断
  - 交互结果：出现在机器右键事件中，写 true 可让右键返回 SUCCESS

## 三、事件 + 节点组合示例（不用 KJS）

下面给出一个新人最常用的模板：**放置默认暂停 + 右键启停 + 取消控制配方**。它覆盖了 80% 的“事件怎么用”的理解方式。

### 目标

- 用 customData.paused 作为开关
- A：放置写 paused=true
- B：右键取反 paused
- C：开始前若 paused=true 则取消（阻止开始）
- D：运行中若 paused=true 则取消（立刻停机）

### 关键套路

- **“先改数据，再让系统逻辑读数据”**：A/B 负责写 customData，C/D 负责读 paused 并取消。
- **“拦截开始”和“中断运行”是两件事**：C 解决开始，D 解决立即停。

## 四、新人排错清单（按出现频率排序）

1. **节点有数据线但没有触发线**：写入类节点不会执行。
2. **复合标签写入后没有把输出写回机器**：必须接“设置自定义数据.数据”。
3. **读到的 paused 永远是 false**：tag 里从没写过 key；先确保 A/B 写入成功。
4. **右键无法“切换”**：异或.b 没接“常量 true”（布尔值开关点亮且不接输入）。
5. **暂停后不停机**：只做了 C 没做 D；D 用于立即中断正在运行的配方。
