# 节点图使用教程（mbd2thread / MBD2）

本文面向 MBD2 编辑器里的“节点图（Graph Processor）”，以本项目的中文显示为准（节点名/端口名会显示为中文）。

## 给新手的三条规则（先看这个）

1. **先找触发起点**：每张事件图里都有“触发开始”，从它的“已触发”端口拉出黄色线，连到你希望按顺序执行的节点。
2. **把数据线接完整**：写入类节点（例如“复合标签写入”“设置自定义数据”“设置机器状态”）除了触发线，还必须把它需要的数据端口接满，否则触发了也不会产生效果。
3. **用 Debug 验证**：调试面板打开“显示调试信息”，先验证你要的字段（例如 paused）有没有变，再验证配方状态（IDLE/WORKING）是否符合预期。

## 核心概念

- **事件图**：由某个机器事件触发的一张图，例如“机器放置事件”“机器右键事件”“配方开始前事件”等。
- **触发线（黄色）**：表示执行顺序。通常从“触发开始 → 已触发”开始，把它连到需要按顺序执行的节点（如“复合标签写入”“设置自定义数据”“设置机器状态”）。
- **数据线（彩色）**：传递数据。不同类型的端口颜色不同（例如机器、NBT 标签、布尔值、字符串等）。
- **参数节点（参数）**：用于读取事件参数（Get）或写回事件参数（Set）。  
  - 可取消事件会额外提供一个可写参数：**取消**（Boolean）。
  - “机器右键事件”提供一个可写参数：**交互结果**（Boolean，true=SUCCESS，false=PASS）。
- **自定义数据（customData）**：机器上持久化存储的一段 NBT（CompoundTag）。适合保存开关、模式、计数器等运行时状态。

## 不用 KubeJS：本教程只用“机器/逻辑/Minecraft数据”等内置节点

本文不使用 “KubeJS / 执行函数” 节点。你只需要：

- 机器分组：机器信息、设置自定义数据、设置机器状态
- Minecraft 数据分组：复合标签 / 复合标签读取 / 复合标签写入
- 值分组：布尔值、字符串
- 逻辑分组：异或、选择（可选）

## 常用事件速查（机器事件）

- 机器放置事件：放下机器时触发。
- 机器右键事件：玩家右键机器时触发。
- 配方开始前事件（可取消）：配方将要开始时触发，写入“取消=true”可以阻止本次开始。
- 配方开始事件（可取消）：配方正在工作 tick 中触发，写入“取消=true”可立刻中断正在运行的配方（更“立即”的停机体验）。

## 事件分步参考（新人按这个顺序学）

1. **先学“写 customData”**：机器放置事件（A）只做一件事：写 `paused=true`。
2. **再学“切换 customData”**：机器右键事件（B）只做一件事：把 `paused` 取反。
3. **最后学“用事件取消控制配方”**：配方开始前事件（C）阻止开始；配方开始事件（D）立即中断。

## 常用节点速查

- 触发开始：事件图的起点（输出端口“已触发”）。
- 机器信息：输入“机器”，输出“自定义数据 / 状态 / 配方状态 / 世界 / 坐标 ……”。
- 复合标签：把输入转换成“复合标签（CompoundTag）”，可用于兜底空标签。
- 复合标签读取：从 CompoundTag 里按 key 读取值（支持 BOOL/NUMBER/STRING/COMPOUND/LIST）。
- 复合标签写入：把 key/value 写入到一个 tag 的副本里（输出新的 tag）。
- 设置自定义数据：把 tag 写回到机器的 customData。
- 设置机器状态：把机器的“状态机状态名”切换到指定字符串（如 working / waiting / suspend）。只有状态机里存在该状态名才会生效。
- 布尔值：一个布尔常量/转换节点。**圆形开关**就是它的内部值（true/false）；当“输入”没连线时输出内部值。
- 异或：`a XOR b`。最常用用法是 `paused XOR true` 实现取反。

## 节点用法详解（新人最常卡的几个）

### 1) 布尔值（常量 true/false）

- 只有一个圆形开关时：这就是“常量模式”。  
  - **输入口没连线** → 输出等于开关值（true/false）。
  - **输入口有连线** → 输出等于输入转换后的布尔值，开关会被同步成当前输出。
- 最常见用法：当作“常量 true”，把开关点亮，并且不要连接输入线。

### 2) 复合标签读取（读 paused）

- 类型选 BOOL，key 填 `paused`。
- 如果 tag 里不存在该 key，会读到默认值 false。
- 所以必须先用 A/B 之一把 `paused` 写进 customData，读取才有意义。

### 3) 复合标签写入（写 paused）

- 需要三样东西：tag、key、value，再加一条触发线。
- 它会输出一个“新的 tag”（副本），所以后面写回机器时要接“复合标签写入.输出”，不要接原始 tag。

### 4) 参数（取消 / 交互结果）

- **取消**：只在“可取消事件”里出现。写 true 就相当于 `event.setCanceled(true)`。
- **交互结果**：只在“机器右键事件”里出现。写 true 会让右键交互返回 SUCCESS（更容易“吃掉”这次右键）。

## 实战：放置默认暂停 + 右键启停 + 工作发光 15

目标：

- 放下机器后默认暂停（不允许配方开始）。
- 右键一次开始工作，再右键一次停止工作（立即停机）。
- 工作状态发光 15（由状态机 working 的光照=15 实现）。

实现思路：

- 在机器 customData 里维护一个布尔字段：`paused`（仅仅是我们自定义的名字，不是 MBD2 内置字段）。
- `paused=true` 表示不允许工作；`paused=false` 表示允许工作。
- 用事件图把 `paused` 写入/切换，并用“可取消事件”的 **取消** 参数拦截/中断配方。

### 0) 先做状态机光照（非节点图）

在机器定义的状态机里：

- 确保存在 `working` 状态，并把它的光照设为 15。
- 其它状态光照按需设为 0。

说明：

- MBD2 会在配方逻辑状态切换时，自动尝试把机器状态切到 `working / waiting / suspend` 等同名状态。
- 若你没有在状态机里创建 `suspend`，那么“设置机器状态=suspend”会被忽略，但不影响“暂停逻辑”的正确性（暂停逻辑由 cancel 控制）。

### A) 机器放置事件：写 paused=true（默认暂停）

事件：**机器放置事件**

节点与连线：

1. 参数（machine）→ 机器信息.机器
2. 机器信息.自定义数据 → 复合标签.输入
3. 触发开始.已触发 → 复合标签写入.触发
4. 复合标签.输出 → 复合标签写入.标签
5. 布尔值（开关点亮=true，且不接输入）→ 复合标签写入.value
6. 复合标签写入：key 填 `paused`
7. 复合标签写入.已触发 → 设置自定义数据.触发
8. 参数（machine）→ 设置自定义数据.机器
9. 复合标签写入.输出 → 设置自定义数据.数据

可选（只用于外观表现）：

- 设置机器状态：字符串 `"suspend"` → 设置机器状态.状态；设置自定义数据.已触发 → 设置机器状态.触发；参数（machine）→ 设置机器状态.机器

验证：

- 重新放置机器后查看 NBT，应看到 `customData:{paused:1b}`。

### B) 机器右键事件：paused 取反（启停开关）

事件：**机器右键事件**

目标：

- 每次右键：`paused = paused XOR true`（true/false 来回切换）
- 顺便把“交互结果”写成 true，避免右键被其它交互吞掉（可选但推荐）

节点与连线：

1. 参数（machine）→ 机器信息.机器
2. 机器信息.自定义数据 → 复合标签.输入
3. 复合标签.输出 → 复合标签读取.标签（类型=BOOL，key=`paused`）
4. 复合标签读取.输出 → 布尔值（转换）.输入 → 异或.a
5. 布尔值（常量 true，开关点亮=true，且不接输入）→ 异或.b
6. 触发开始.已触发 → 复合标签写入.触发
7. 复合标签.输出 → 复合标签写入.标签
8. 异或.out → 复合标签写入.value（key=`paused`）
9. 复合标签写入.已触发 → 设置自定义数据.触发
10. 参数（machine）→ 设置自定义数据.机器
11. 复合标签写入.输出 → 设置自定义数据.数据

可选（推荐）：

- 参数（交互结果）（Set）← 布尔值（常量 true）

验证：

- 右键两次后查看 NBT：`customData.paused` 应在 `0b ↔ 1b` 间切换。

### C) 配方开始前事件：paused=true 时阻止“下一次开始”

事件：**配方开始前事件（可取消）**

节点与连线（与 A/B 同样读取 paused）：

1. 参数（machine）→ 机器信息.机器
2. 机器信息.自定义数据 → 复合标签.输入 → 复合标签.输出 → 复合标签读取（BOOL, key=`paused`）.标签
3. 复合标签读取.输出 → 布尔值（转换）.输入
4. 布尔值.输出 → 参数（取消）（Set）.输入

效果：

- paused=true：阻止配方进入 WORKING（不消耗/不发光/不工作）
- paused=false：允许开始

### D) 配方开始事件：paused=true 时立刻中断“正在运行中”

事件：**配方开始事件（可取消）**

连法与 C 完全相同（按 paused 写入“取消”）。

为什么需要它：

- 仅做 C 时，“暂停”更多是阻止下一次开始；已经 WORKING 的配方不一定立即停。
- 加上 D 后，paused=true 会在工作 tick 中被 cancel，从而立刻 interrupt（停机体验更直觉）。

## 常见问题排查

- 右键一次能开，第二次关不掉：
  - 优先检查 B：异或.b 是否确实是“常量 true”（布尔值开关点亮且输入没连线）。
  - 再检查 D 是否存在并写了“取消=paused”（用于立刻停机）。
- `suspend` 不生效：
  - 你没在机器状态机里创建 `suspend` 状态。暂停逻辑依旧可用，但外观不会切到 suspend。
- paused 一直读到 false：
  - 说明 A/B 没写成功或写到了别的 tag；检查“设置自定义数据”的数据输入是否接的是“复合标签写入”的输出。
